设计思想与需求背景：
- d1vai 在最近 25 次更新里，核心变化集中在「模型选择链路」和「workspace 就绪门控」：创建项目前选模型、聊天时可切模型、workspace 未就绪时延迟加载模型、模型切换时禁止发送、并把用户缓存模型自动同步到后端。
- d1vai_app 目前已具备 workspace 预热、消息队列和重连基础能力，但缺少完整的模型配置闭环（读取/切换/缓存/同步/发送时携带），导致移动端与 Web 行为不一致。
- 本次目标：在不引入仅 Web 场景逻辑的前提下，把 d1vai_app 对齐到同一套「可感知状态 + 可恢复流程 + 可追踪反馈」体验；每完成一个小功能都先跑检查，再在下面勾选。
